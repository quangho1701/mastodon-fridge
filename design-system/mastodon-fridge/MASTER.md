# Design System Master File

> **Priority:** `DESIGN.md` at the repository root is the highest-priority design authority for Mastodon Fridge.
> It is tailored to Purdue Fort Wayne students and the PFW brand, so it overrides any generated or generic design-system guidance.

---

## Source of Truth

Always start with `DESIGN.md` before making UI, layout, spacing, color, typography, component, or microcopy decisions.

If another generated design-system file conflicts with `DESIGN.md`, follow `DESIGN.md`.

If a page-specific file is added under `design-system/mastodon-fridge/pages/`, use it only as supplemental guidance. Page files may add implementation details, but they must not replace or contradict `DESIGN.md`.

---

## Project Identity

**Project:** Mastodon Fridge  
**Audience:** Purdue Fort Wayne students  
**Design foundation:** PFW Black and Gold, mobile-first campus utility, tactile communal-fridge metaphor

---

## Required Design Baseline

Use the root `DESIGN.md` for:

- PFW-rooted color palette, including Summit Gold for actions and Golden for identity moments
- United Sans Cond and Acumin Pro typography
- 4px spacing system and 44px minimum touch targets
- Light-mode shadows and dark-mode border-only depth
- Fridge-inspired tactile components such as magnets, flyers, sticky notes, polaroids, and pinned items
- Brand voice for PFW students, including when to use "Dons" versus "Mastodons"
